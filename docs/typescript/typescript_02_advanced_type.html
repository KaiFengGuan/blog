Typescript高级类型

GKFTech 2022/5/6 TypeScript

Typescript 是 JavaScript 的一个超集，而且本质上向这个语言添加了可选的静态类型和基于类的面向对象编程。

# 1 联合类型

如果希望一个变量可以支持多种类型，就可以用联合类型（union types）来定义。

例如，一个变量既支持 number 类型，又支持 string 类型，就可以这么写：

let num: number | string

num = 8
num = 'eight'

num.toString();	
// num.length   // 报错，因为number没有length属性

# 2 交叉类型

交叉类型是将多个类型合并为一个类型。

如：Person 有 name 和 age 的属性，而 Student 在 name 和 age 的基础上还有 grade 属性，就可以这么写：

interface Person {
    name: string;
    age: number;
}
type Student = Person & { grade: number };

与联合类型的区别：

  • 联合类型可以取几种类型中的任意一种
  • 交叉类型是把几种类型合并起来

# 3 类型别名

类型别名会给一个类型起个新名字。

类型别名会给一个类型起个新名字。 类型别名有时和接口很像，但是可以作用于原始值，联合类型，元组以及其它任何你需要手写的类型。 -- TS 文档

type Name = string;
type NameResolver = () => string;
type NameOrResolver = Name | NameResolver;
function getName(n: NameOrResolver): Name {
    if (typeof n === 'string') {
        return n;
    }
    else {
        return n();
    }
}

type和interface用法功能都很像，两者由什么异同？

  • 相同点：

    • 都可以定义一个对象或函数
    • 都允许继承（interface继承使用extends关键字，type继承使用交叉类型）

  • 不同点：

    • 目的上：interface是接口，是TS设计出来用于定义对象类型的，可以对对象的形状进行描述；type 是类型别名，用于给各种类型定义别名，让 TS 写起来更简洁、清晰。

    • 使用上：type 可以声明基本类型、联合类型、交叉类型、元组，interface 不行；interface可以合并重复声明，type 不行。

      // 合并重复声明
interface Person { name: string }
interface Person { age: number }
const person: Person = {
    name: 'lin',
    age: 18
}

// type会报错
type Person = { name: string }
type Person = { age: number }	// 报错：Duplicate identifier 'Person'

  • 题外话：

    • 其实本不该把这两个东西拿来做对比，他们俩是完全不同的概念。interface 是接口，用于描述一个对象。type 是类型别名，用于给各种类型定义别名，让 TS 写起来更简洁、清晰。
    • 只是有时候两者都能实现同样的功能，才会经常被混淆
    • 平时开发中，一般使用组合或者交叉类型的时候，用 type。一般要用类的 extends 或 implements 时，用 interface。

# 4 类型保护

类型保护（Type Guards）：可以使用 typeof 关键字判断变量的类型。

如果有一个 getLength 函数，入参是联合类型 number | string，返回入参的 length：

// 报错，因为number没有length属性
function getLength(arg: number | string): number {
    return arg.length
}

// 正常，typeof类型保护将变量缩减为那个具体的类型
function getLength(arg: number | string): number {
    if(typeof arg === 'string') {
        return arg.length
    } else {
        return arg.toString().length
    }
}

但是TypeScript并不会阻止你与其它字符串比较，语言不会把那些表达式识别为类型保护。

# 5 类型断言

上文类型保护的例子也可以使用类型断言来解决。

类型断言语法：

value as type
1

使用类型断言来告诉 TS，我（开发者）比你（编译器）更清楚这个参数是什么类型，你就别给我报错了：

function getLength(arg: number | string): number {
    const str = arg as string
    if (str.length) {
        return str.length
    } else {
        const number = arg as number
        return number.toString().length
    }
}

# 6 字面量类型

字符串字面量类型允许你指定字符串必须的固定值。

在实际应用中，字符串字面量类型可以与联合类型，类型保护和类型别名很好的配合。 通过结合使用这些特性，你可以实现类似枚举类型的字符串。

字面量类型也支持数字字面量类型。

type ButtonSize = 'mini' | 'small' | 'normal' | 'large'

type Sex = '男' | '女'
const sex: Sex = '不男不女'	// 报错
const sex: Sex = '男';

type Num = 1 | 2 | 3;
const num: Num = 2;

# 7 索引类型

使用索引类型，编译器就能够检查使用了动态属性名的代码。

如：从对象中抽取一些属性然后拼接成数组

const userInfo = {
  name: 'lin',
  age: '18',
}

function getValues(userInfo: any, keys: string[]) {
  return keys.map(key => userInfo[key])
}

// 抽取指定属性的值
console.log(getValues(userInfo, ['name','age']))  // ['lin', '18']
// 抽取obj中没有的属性:
console.log(getValues(userInfo, ['sex','outlook']))  // [undefined, undefined]

可以看到，虽然obj 中并不包含 sex 和 outlook 属性，但 TS 编译器并未报错。

这种情况需要使用TS索引类型进行约束，实现动态属性检查。

function getValues<T, K extends keyof T>(userInfo: T, keys: K[]): T[K][] {
    return keys.map(key => userInfo[key])
}

getValues(userInfo, ['sex','outlook'])	// 报错
1
2
3
4
5
  • 定义泛型 T、K，用于约束 userInfo 和 keys
  • 为 K 增加一个泛型约束，使 K 继承 userInfo 的所有属性的联合类型，即K extends keyof T
  • 通过 keyof 关键字获取 interface 中的所有键，返回类型为联合类型
  • T[K] 表示接口T的属性K所代表的类型

# 8 映射类型

类型映射用来将一个类型映射成另一个类型。使用in操作符对联合类型遍历实现。

# 8.1 Partial

Partial<T>将T的所有属性映射为可选的。

例如：

interface IPerson {
    name: string
    age: number
}

type IPartial = Partial<IPerson>
// IPartial的name和age变成可选属性
1
2
3
4
5
6
7

实现原理：

type Partial<T> = {
    [P in keyof T]?: T[P]
}
1
2
3
  • [P in keyof T]遍历T上的所有属性
  • ?:设置属性为可选的
  • T[P]设置类型为原来的类型

# 8.2 Readonly

Readonly<T>将T的所有属性映射为只读的。

例如：

interface IPerson {
    name: string
    age: number
}

type Readonly = Readonly<IPerson>
// IPartial的name和age变成只读属性
1
2
3
4
5
6
7

实现原理：

type Readonly<T> = {
    readonly [P in keyof T]: T[P]
}
1
2
3
  • [P in keyof T]遍历T上的所有属性
  • readonly设置属性为只读的
  • T[P]设置类型为原来的类型

# 8.3 Pick

Pick用于抽取对象子集，挑选一组属性并组成一个新的类型。

如：

interface IPerson {
    name: string
    age: number
    sex: string
}

type IPick = Pick<IPerson, 'name' | 'age'>
let p1: IPick = {
    name: 'li',
    age: 18
}

  • 第一个参数T，表示要抽取的目标对象
  • 第二个参数K，具有一个约束：K一定要来自T所有属性字面量的联合类型

# 9 条件类型

T extends U ? X : Y 
//若类型 T 可被赋值给类型 U,那么结果类型就是 X 类型,否则就是 Y 类型
1
2

Exclude 和 Extract 的实现就用到了条件类型。

# 9.1 Exclude

Exclude 意思是不包含，Exclude<T, U> 会返回 联合类型 T 中不包含 联合类型 U 的部分。

type Test = Exclude<'a' | 'b' | 'c', 'a'>
1

实现原理：

type Exclude<T, U> = T extends U ? never : T
1

# 9.2 Extract

Extract<T, U>提取联合类型 T 和联合类型 U 的所有交集。

type Test = Extract<'key1' | 'key2', 'key1'>
1

实现原理：

type Extract<T, U> = T extends U ? T : never
1

# 10 工具类型

为了方便开发者使用， TypeScript 内置了一些常用的工具类型。上文介绍的索引类型、映射类型和条件类型都是工具类型。

除了上文介绍的，再介绍一些常用的，毕竟工具函数遇到了去查就行，死记硬背就太枯燥了，熟能生巧，写多了自然就熟悉了。

# 10.1 Omit

Omit<T, U>从类型 T 中剔除 U 中的所有属性。

interface IPerson {
    name: string
    age: number
}

type IOmit = Omit<IPerson, 'age'>
// type IOmit = {
//     name: string;
// }
1
2
3
4
5

7
8
9

实现原理：

type Omit<T, K extends keyof any> = 
Pick<T, Exclude<keyof T, K>>
1
2

Pick用于挑选一组属性并组成一个新的类型，Omit 是剔除一些属性，留下剩余的，他们俩有点相反的感觉。那么就可以用 Pick 和 Exclude 实现 Omit。

当然也可以不用 Pick 实现：

type Omit2<T, K extends keyof any> = {
    [P in Exclude<keyof T, K>]: T[P]
}

# 10.2 NonNullable

用来过滤类型中的 null 及 undefined 类型。

type T0 = NonNullable<string | number | undefined>;	// string | number
type T1 = NonNullable<string[] | null | undefined>; // string[]
1
2

实现原理：

type NonNullable<T> = T extends null | undefined ? never : T
1
  • never表示一个不存在的类型
  • never与其他类型的联合后，为其他类型

# 10.3 Parameters

Parameters 获取函数的参数类型，将每个参数类型放在一个元组中。

type T1 = Parameters<() => string>  // []

type T2 = Parameters<(arg: string) => void>  // [string]

type T3 = Parameters<(arg1: string, arg2: number) => void> // [arg1: string, arg2: number]
1
2
3
4
5

实现原理：

type Parameters<T extends (...args: any) => any> = 
T extends (...args: infer P) => any ? P : never
1
2

在条件类型语句中，可以用 infer 声明一个类型变量并且对它进行使用。

  • Parameters首先约束参数T必须是个函数类型
  • 判断T是否是函数类型，如果是则使用infer P暂时存一下函数的参数类型，后面的语句直接用 P 即可得到这个类型并返回，否则就返回never

# 10.4 ReturnType

ReturnType 获取函数的返回值类型。

type T0 = ReturnType<() => string>  // string

type T1 = ReturnType<(s: string) => void>  // void
1
2
3

ReturnType 原理

/**
 * Obtain the return type of a function type
 */
type ReturnType<T extends (...args: any) => any> = 
T extends (...args: any) => infer R ? R : any
1
2
3
4
5

懂了 Parameters，也就懂了 ReturnType，

  • ReturnType首先约束参数T必须是个函数类型
  • 判断T是否是函数类型，如果是则使用infer R暂时存一下函数的返回值类型，后面的语句直接用 R 即可得到这个类型并返回，否则就返回any
