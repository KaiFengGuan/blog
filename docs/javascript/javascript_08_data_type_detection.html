JS数据类型检测的四种办法

GKFTech 2022/3/23 JavaScript

主要有4种方式：

  1. typeof
  2. instanceof
  3. constructor
  4. Object.prototype.toString.call()

# 1 typeof

typeof运算符返回一个字符串，表示操作数的类型。

语法：

typeof operand
typeof(operand)
1
2

例子：

typeof 3  // "number"
typeof "" // "string"
typeof true  // "boolean"
typeof 12n  // "bigint" ES2020
typeof Symbol()  // "symbol"

typeof undefined  // "undefined"
typeof null  // "object"

typeof {}  // "object"
typeof []  // "object"
typeof function (){} // "function"
typeof class C {}  // "function"

  • 但是遇到引用数据类型（如：Array）是不起作用的
  • typeof null==="object"是历史遗留问题

# 2 instanceof

instanceof运算符测试构造函数的属性是否 出现prototype在对象的原型链中的任何位置。

利用这个特性也可以检测数据类型。

语法：

object instanceof constructor
1

示例：

new Number(1) instanceof Number  // "true"
new String("") instanceof String  // "true"
new Boolean("") instanceof Boolean  // "true"

({}) instanceof Object  // "true"
[] instanceof Array     // "true"
(function(){}) instanceof Function  // "true"
(class C{}) instanceof Function     // "true"

总结：

  • 对于基础类型的判断，需要转换为其对应的对象后才能判断（因为 instanceof 不能用于判断基本数据类型，只能用于判断引用数据类型）
  • 对于引用类型数据，可以得出正确结果
  • Symbol和BigInt不能作为构造函数，无法判断
  • 不能判断null和undefined类型

# 3 constructor

**constructor**是一种用于创建和初始化class创建的对象的特殊方法。

语法：

constructor([arguments]) { ... }
1

示例：

(1).constructor === Number
("").constructor === String
(true).constructor === Boolean
(12n).constructor === BigInt
(Symbol()).constructor === Symbol

({}).constructor === Object
([]).constructor === Array
(function(){}).constructor === Function
(class C {}).constructor === Function

总结：

  • 基本上可以检测出基本数据类型和引用数据类型
  • 无法判断null和undefined，因为他们没有构造函数

# 4 Object.prototype.toString.call

终极解决办法：使用Object.prototype.toString.call。（jQuery采用的数据类型检测方式）

示例：

Object.prototype.toString.call(3)         // '[object Number]'
Object.prototype.toString.call("")        // '[object String]'
Object.prototype.toString.call(true)      // '[object Boolean]'
Object.prototype.toString.call(12n)       // '[object BigInt]'
Object.prototype.toString.call(Symbol())  // '[object Symbol]'

Object.prototype.toString.call(undefined) // '[object Undefined]'
Object.prototype.toString.call(null)      // '[object Null]'

Object.prototype.toString.call({})        // '[object Object]'
Object.prototype.toString.call([])        // '[object Array]'
Object.prototype.toString.call(function (){})  // '[object Function]'
Object.prototype.toString.call(class C {})     // '[object Function]'

总结：

  • 使用call或者apply方法来调用的是Object.prototype上的toString方法（改变执行上下文），避免toString被改写
  • 对于所有基本的数据类型都能进行判断，包括null和undefined。
