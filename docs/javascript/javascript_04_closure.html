# 1 闭包

根据前面讲到的作用域和作用域链可以知道，外层作用域无法获取内层作用域下的变量，同样在不同的函数作用域中也是不能相互访问彼此变量，那么当我们想在一个函数内部也有限权访问另一个函数内部的变量该怎么办呢？我们可以使用闭包。

# 1.1 闭包定义

闭包是指有权访问另一个函数作用域中的变量的函数。创建闭包的常见方式就是在一个外部函数内部创建另一个函数并将其返回，闭包包含了外部函数在创建时作用域中的所有变量。

通过一个例子和上一篇讲的作用域及作用域链来看看闭包是如何起作用的：

function test1 () {
  function test2 () {
    var b = 0;
    console.log(a)
  }
  var a = 1;
  return test2;
}

var c = 3;
var test3 = test1();
test3();

无论什么时候在函数中访问一个变量时，就会从作用域链中搜索具有相应名字的变量。一般来讲，当函数执行完毕后，局部活动对象（AO）就会被销毁，内存中仅保存全局作用域（全局执行环境的变量对象）。但是，闭包的情况又有所不同，由于作用域仍保留着对AO的引用，因此该对象不会被销毁，所以我们仍能访问到AO中的变量，使得外部作用域也能访问到内部作用域中的变量。

# 1.2 闭包的优缺点

(1) 优点：

  • 保护函数内的变量安全，实现封装，防止变量流入其他环境发生命名冲突；
  • 在内存中维持一个变量，可以做缓存；
  • 将上级作用域的引用保存下来，实现方法或属性的私有化。

(2) 缺点：

  • 闭包会导致原来产生的作用域链不释放，过渡使用闭包可能会导致内存泄露，或加载过慢。

# 1.3 闭包应用案例

(1) 模仿块级作用域

比如我们可以使用闭包能使下面的代码按照我们预期的进行执行（每隔1s打印 0,1,2,3,4）。使用立即执行函数尽量避免往全局作用域中添加变量和函数。

for(var i = 0; i < 5; i++) {
    (function(j){
        setTimeout(() => {
            console.log(j);
        }, j * 1000);

(2) 私有变量

JavaScript中没有私有成员的概念，所有属性都是公有的。但是有私有变量的概念，任何在函数中定义的变量，都可以认为是私有变量，因为在函数的外部不能访问这些变量。私有变量包括函数的参数，局部变量和函数内部定义的其他函数。通过在函数内部创建一个闭包，闭包可以通过自己的作用域链就可以访问这些变量，由此可以创建用于访问私有变量的公有方法（也称为特权方法）。

function MyObject() {
    // 私有变量和私有函数
    var privateVariable = 10;
    function privateFunction() {
        return false;
    }
    // 特权方法
    this.publicMethod = function() {
        privateVariable++;
        return privateFunction;
    }
}

(4) Vue源码中的cached函数结果缓存

Vue源码中经常能看到下面这个cached函数(接收一个函数，返回一个函数)。

/**
* Create a cached version of a pure function.
*/
function cached (fn) {
    var cache = Object.create(null);
    return (function cachedFn (str) {
        var hit = cache[str];
        return hit || (cache[str] = fn(str))
    })
}

# 2 立即执行函数

立即执行函数（Immediately Invoked Function Expression, IIFE），在定义函数的时候直接执行。这里不是申明函数而是一个函数表达式。

一定是表达式（不论是不是匿名函数）才能被执行符号立即执行，使用括号包裹函数可以将函数转为表达式，实际上使用表达式的话函数名就没什么意义了，因此一般忽略函数名使用匿名函数。

// 写法 1
(function (args) {
  /* 程序逻辑 */
})(args);

// 写法 2
(function (args) {
  /* 程序逻辑 */
}(args));
