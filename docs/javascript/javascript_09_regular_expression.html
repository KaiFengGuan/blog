学学正则表达式

GKFTech 2022/5/2 JavaScript正则表达式

正则表达式常见字符串匹配的场景，如各种表单验证，Vue模板解析也用到了正则表达式...... 正则表达式是匹配模式，要么匹配字符，要么匹配位置。

# 1 位置匹配

# 1.1 啥是位置？

位置可以理解为相邻字符之间的位置。可以和空字符串进行类比, 字符的首尾、间隙都可以用空字符串进行连接。

'hello' === '' + 'h' + '' + 'e' + '' + 'l' + '' +  'l' + '' + 'o' + ''
1

# 1.2 有哪些位置？

符号 含义
^ 脱字符，匹配行的开头
$ 美元符，匹配行的结尾
\b 单词的边界，具体讲有三点规则：
（1）\w和\W之间的位置；
（2）^与\w之间的位置；
（3）\w与$之间的位置
\B 非单词的边界，与\b相反：
（1）\w和\w之间的位置；
（2）\W和\W之间的位置；
（3）^与\W之间的位置；
（4）\W与$之间的位置

示例理解：

// ^
'hello'.replace(/^/, '😄')		// 😄hello

// $
'hello'.replace(/$/, '😄')		// hello😄

// \b
'hello.mp4'.replace(/\b/g, '😄')	// '😄hello😄.😄mp4😄'

// \B
'hello.mp4'.replace(/\B/g, '😄')	// 'h😄e😄l😄l😄o.m😄p😄4'

# 1.3 向前断言与向后断言

符号 含义
(?=p) 向前断言：符合p子模式前面的位置
(?!p) 向前否定断言：与正向正预查相反，可以理解为(?=p)匹配到的位置之外的位置都属于(?!p)
(?<=p) 向后断言：符合p子模式后面（注意(?=p)表示的是前面）的那个位置。
换句话说是，有一个位置，其前面的部分需要满足p子模式。
(?<!p) 向后否定断言：可以理解为(?<=p)匹配到的位置之外的位置都是属于(?<!p)的
// (?=p)
'hello.mp4'.replace(/(?=ll)/, '😄')	// 'he😄llo.mp4'

// (?!p)
// 除了匹配到 (?=ll) 前的位置，其他的都匹配到了
'hello.mp4'.replace(/(?!ll)/g, '😄') // '😄h😄el😄l😄o😄.😄m😄p😄4😄'

// (?<=p)
'hello.mp4'.replace(/(?<=ll)/g, '😄')	// 'hell😄o.mp4'

// (?<!p)
// 除了匹配到 (?=ll) 后的位置，其他的都匹配到了
'hello.mp4'.replace(/(?<!ll)/g, '😄')	// '😄h😄e😄l😄lo😄.😄m😄p😄4😄'

# 2 字符匹配

# 2.1 通过量词实现模糊匹配（横向）

符号 含义
+ 重复零次或更多次
* 重复一次或更多次（至少有一次）
? 重复零次或一次（可有可无）
{n} 重复 n 次
{m, n} 至少m次，至多m次
{n, } 至少n次

# 2.2 通过字符组实现模糊匹配（纵向）

在中括号[]里面是用短横线-连接（左边的要小于右边的，不然会报错）

表达式 说明
[abc] 匹配方括号内的任意字符
[^abc] 匹配不在方括号内的任意字符
[0-9] 匹配任何从0至9的数字

字符组也有简写形式：

符号 说明
\d 匹配数字
\D 匹配非数字
\w [0-9a-zA-Z]
\W [^0-9a-zA-Z]
\s [\t\v\n\r\f]：匹配空字符
\S [^\t\v\n\r\f]：匹配非空字符
. 匹配任何字符

# 2.3 贪婪匹配 VS 惰性匹配

正则本身是贪婪的，即尽可能多的匹配符合模式的字符串。在量词后加上?符号，即变成惰性匹配，尽可能少地匹配符合模式的字符串。

'123 1234 12345 123456'.match(/\d{2,5}/g)
// ['123', '1234', '12345', '12345']

'123 1234 12345 123456'.match(/\d{2,5}?/g)
// ['12', '12', '34', '12', '34', '12', '34', '56']

# 2.4 分支结构（多选）

一个模式可以实现横向和纵向的模糊匹配，而多选分支可以支持多个子模式任选其一，形式是(p1|p2|p3)，例子：

'good idea, nice try.'.match(/good|nice/g)	// ['good', 'nice']
1
2

注意：分支结构是惰性的，前面的匹配上了，后面的就不再尝试了。如：

'goodbye'.match(/good|goodbye/g)	// ['good']
1

# 2.5 捕获分组 VS 非捕获分组

分组：通过括号提供分组（括号内的正则是一个整体，即提供子表达式）

  • 捕获分组(p) 匹配 p 并且捕获匹配项
  • 非捕获分组(?:p) 匹配 p 不会捕获匹配项。匹配项不能够从结果再次访问
'ababa abbb ababab'.match(/(ab)+/)		// ['abab', 'ab']
'ababa abbb ababab'.match(/(?:ab)+/)	// ['abab']

# 3.1 正则实例方法

（1）regObj.test(string)

对字符串执行搜索，若字符串中存在匹配regObj模式的字符串则返回true，否则返回false。

  • regObj不带全局属性g时：lastIndex属性一直是0
  • regObj带全局属性g时：lastIndex属性为上一次匹配内容最后一个字符的下一个位置。
let str = '1a22bb333ccc'
let reg = /\d+([a-z]+)/

reg.lastIndex	// 0
reg.test(str)	// true
reg.lastIndex	// 0
reg.test(str)	// true
reg.lastIndex	// 0
reg.test(str)	// true
reg.lastIndex	// 0
reg.test(str)	// true

（2）regObj.exec(string)

对字符串执行搜索，如果没有匹配的文本则返回 null，否则返回一个结果数组。

regObj带不带全局属性的表现同test，具体看例子：

reg.lastIndex	// 2
reg.exec(str)	// ['22bb', 'bb']
reg.lastIndex	// 6
reg.exec(str)	// ['333ccc', 'ccc']
reg.lastIndex	// 12
reg.exec(str)	// null

# 3.2 字符串实例方法

（1）string.search(reg)

找出首次匹配项的索引，返回匹配成功的第一个位置，如果没有任何匹配则返回-1。

'abc'.search(/b/)	// 1
1

（2）string.split()

  • 第一个参数：可以是字符串或者正则表达式，它是分隔符；
  • 第二个参数：可选，限制返回数组的最大长度
  • 返回值：数组
'abc-def_mno+xyz'.split(/[-_+]/, 3);	// ["abc", "def", "mno"]
1

（3）string.match(reg)

找到一个或多个正则表达式的匹配，返回数组或者 null。

  • reg不带g时，只查找一次，遇到匹配之后就停止，返回结果和 regObj.exec 方法一致
  • reg带g时，全局检索，如果找到了一个或多个匹配字符串，则返回一个数组（不包含捕获组）
let str = '1a22bb333ccc'
let reg = /\d+([a-z]+)/

str.match(reg)	// ['1a', 'a']

let str = '1a22bb333ccc'
let reg = /\d+([a-z]+)/g

str.match(reg)	// ['1a', '22bb', '333ccc']

（4）string.replace()

  • 第一个参数：可以是字符串或者正则表达式，它的作用是匹配
    • 参数是字符串的话只能进行一次替换
    • 正则表达式的表达能力更强，而且可以全局匹配
  • 第二个参数：可以是字符串或者函数，它的作用是替换
    • 参数是字符串时，可以使用$符号表示相应顺序的捕获组（不要写成模板字符串的形式）
    • 参数是函数时，返回值就是要替换的内容，不指定返回值则默认为undefined
      • 第一个参数 matchStr 是匹配结果
      • 第二个参数以及到倒数第二个参数为止表示捕获组，如果有捕获组，函数的后顺位参数与捕获组一一对应
      • 倒数第二个参数是匹配结果在文本中的位置
      • 最后一个参数是源文本
  • 返回值：替换了之后的新的字符串，原字符串不变

例子：

let str = 'a1b2c3d4e'
let result = str.replace(/(\d)(\w)(\d)/g, (matchStr, g1, g2, g3, idx, originStr) => {
    return g1 + g3
})
//'a12c34e'

# 4 常用正则匹配示例

# 4.1 数字千分位切割

将123456789转化为123,456,789

  • 第一步：先把后面第一个逗号弄出来

    '123456789'.replace(/(?=\d{3}$)/g, ',')	// '123456,789'
    1

  • 第二步：多个组一起匹配

    用小括号将\d{3}括起来，表示一个分组，后面的+号表示重复一次或多次

    '123456789'.replace(/(?=(\d{3})+$)/g, ',')	// ',123,456,789'
    1

  • 第三步：去掉首位的括号

    (?!^)表示不匹配开头

    '123456789'.replace(/(?!^)(?=(\d{3})+$)/g, ',')	// '123,456,789'
    1

# 4.2 手机号3-4-4分割

思路跟数字千分位切割类似：

'15722334455'.replace(/(?=(\d{4})+$)/g, '-')	// '157-2233-4455'
1

# 4.3 字符串驼峰化

  • foo Bar => fooBar
  • foo-bar---- => fooBar
  • foo_bar__ => fooBar

分析：

  • 每个单词的前面都有0个或者多个-、空格、_；
  • -、空格、_后有可能不跟任何东西
const toCamelCase = (string) => {
  const exp = /[-_\s]+(.)?/g;
  return string.replace(exp, (match, char) => {
    return char ? char.toUpperCase() : '';
  });
}

let strs = ['foo Bar', 'foo-bar---', 'foo_bar__'];
console.log( strs.map(str => toCamelCase(str)) )

# 4.4 将字符串首字母转化为大写

如：hello world 转为为Hello World

分析：找到单词的首字母然后将其转化为大写字母就可以，单词前面可能是开头也可能是多个空格。

const toUpper = (string) => {
  const exp = /(^|\s+)\w/g;
  return string.toLowerCase().replace(exp, match => match.toUpperCase());
}
toUpper('hello word! HA-Ha-hA-ha......')	// Hello Word! Ha-ha-ha-ha......

# 4.5 通过name获取url query参数

name可能处于3个位置：

  • 紧跟问号
  • 在最后的位置
  • 在两个其他参数中间

因此只要处理三个地方：

  • name前面只能是 ? 或者 &；
  • value的值是除了 & 以外的任意字符；
  • value后面只能是 & 或者结束位置 $。
function getURLParam(name) {
    const regex = new RegExp(`[?&]${name}=([^&]*)(&|$)`);
    const match = window.location.search.match(regex);
    // 取 match[1] 是取第 1 个捕获组
    return match ? decodeURIComponent(match[1]) : '';
}

上面是获取单个参数的场景，还有一种情况是获取所有参数：

function getURLParam(url) {
    const params = {};
    // ([^?&]+) 意思是匹配非 ? 及 & 开头的非空字符串
    // (?==) 意思是非获取匹配 = ，向前断言，获取不带等号结尾的 key 值
	const keys = url.match(/([^?&]+)(?==)/g);
    // (?<==) 意思是非获取匹配 = ，向后断言，以便获取不带等号开头的 value 值
    // ([^&]*) 意思是匹配非 ? 及 & 结尾的字符串（包含空字符串情况，因为有的 value 值会为空）
	const values = url.match(/(?<==)([^&]*)/g);
	for(const index in keys){
		params[keys[index]] =  values[index];
	}
	return params;
}

# 4.7 提取连续重复的字符

币安原题：将有重复的字符提取出来，例如12323454545666，提取[ '23', '45', '6' ]

分析过程：

  • 使用分组(.+)表示n个字符串可以重复
  • 使用反向引用来引用第一次匹配到的分组()括起来的部分
  • 反向引用的字符也可以是多个，因此是\1+。
const collectRepeatStr = (str) => {
  let repeatStrs = []
  const repeatRe = /(.+)\1+/g
  // 很多时候replace并不是用来做替换，而是做数据提取用
  str.replace(repeatRe, ($0, $1) => {
    console.log($0, $1);
    $1 && repeatStrs.push($1)
  })
  
  return repeatStrs
}

# 4.8 模板字符串替换

有一个模板需要替换上面的值

template = '我是，年龄，性别';

分析思路：

  • 这里只要捕获{{}}内的内容就行了
  • 需要注意的是{{}}内可能有0个以上的的空格
function templateToStr(template, data) {
  const regexp = /\{\{\s{0,}(\w+)\s{0,}\}\}/g;
  return template.replace(regexp, (match, char) => data[char]);
}

let template = '我是{{  name}}，年龄{{age }}，性别{{ sex }}';
let data = {
  name: '张三',
  age: 18,
  sex: '男'
}
console.log( templateToStr(template, data) )
// 我是张三，年龄18，性别男
