# 1 HTML5新特性

  • 语义标签：使开发者更方便清晰构建页面的布局，语义化更清晰

    • header、footer、nav、section、article、aside、main等

  • 增强型表单：修改一些新的input输入特性，改善更好的输入控制和验证

    • 新输入类型：color、date、email、tel、search、range
    • 新表单元素：datalist、progress、keygen（提供验证用户的可靠方法）
    • 新表单属性：placehoder、required、pattern、autofocus

  • 视频和音频：<audio>

  • Canvas绘图：使用JS在画布上进行绘图

  • SVG绘图：可伸缩矢量图形，XML格式定义

    Canvas SVG
    大数据量 ✅ ❌
    高交互场景 ✅ ❌
    可导出图片 ✅ ❌
    放大不失真 ❌ ✅
    支持事件处理 ❌ ✅
    文字可编辑 ❌ ✅

  • 地理定位：使用getCurrentPosition()方法来获取用户的位置。以实现LBS服务

    • LBS：Location Based Services，基于位置的服务

  • 拖放API：捉取对象以后拖到另一个位置

  • WebWorker：加载脚本文件运行一个独立于主线程之外的线程

    • 使用Worker类加载一个javascript文件开辟新的线程，起到互不阻塞执行的效果
    • 提供主线程和新线程之间数据交换的接口：postMessage、onmessage。

  • WebStorage：新增的本地存储解决方案之一，拥有5M的存储容量

    • cookie作为HTTP协议的一部分用来处理客户端和服务器的通信，是不可或缺的
    • session正式依赖与实现的客户端状态保持
    • WebSorage的意图在于解决本来不应该cookie做，却不得不用cookie的本地存储，但并不是取代cookie

  • WebSocket：为web客户端和服务端之间提供了一种全双工通信机制

    • 握手阶段采用HTTP协议，默认端口是80和443，建立在TCP协议基础之上，和http协议同属于应用层
    • 协议标识符是ws（如果加密，为wss），如ws://localhost:8023
    • 可以发送文本，也可以发送二进制数据
    • 没有同源限制，客户端可以与任意服务器通信。

# 2 HTML 块级元素 & 行内元素

  1. 行内元素：a、button、img、input、label、img、q、strong、em、cite、等
  • 可以和其他元素处于一行，不用必须另起一行。
  • 元素的高度、宽度及顶部和底部边距不可设置。
  • 元素的宽度就是它包含的文字、图片的宽度，不可改变。
  1. 块级元素：header、footer、aside、article、nav、audio、canvas、details、h1-h6、p、pre、section、table、ul/ol/li、form、等
  • 每个块级元素都是独自占一行。
  • 元素的高度、宽度、行高和边距都是可以设置的。
  • 元素的宽度如果不设置的话，默认为父元素的宽度（父元素宽度100%）。

# 4 defer和async的区别

浏览器在遇到script标签的时候，文档的解析会停止，不再构建document，有时打开一个网页上会出现空白一段时间，浏览器显示是刷新请求状态(也就是一直转圈)，这就会给用户很不好的体验。 defer和async用于解决这个问题。它们都是用于在不阻塞页面文档解析的前提下，控制脚本的下载和执行。

  • defer：用于开启新的线程下载脚本文件，并使脚本在文档解析完成后执行。

    • defer只适用于外联脚本，如果script标签没有指定src属性，只是内联脚本，不要使用defer
    • 如果有多个声明了defer的脚本，则会按顺序下载和执行
    • defer脚本会在DOMContentLoaded和load事件之前执行

  • async：用于异步下载脚本文件，下载完毕立即解释执行代码。

    • 只适用于外联脚本，这一点和defer一致
    • 如果有多个声明了async的脚本，其下载和执行也是异步的，不能确保彼此的先后顺序
    • async会在load事件之前执行，但并不能确保与DOMContentLoaded的执行先后顺序 。

注意：

  • 浏览器对于带有type="module"的<script>，都是异步加载，不会造成堵塞浏览器，即等到整个页面渲染完，再执行模块脚本，等同于打开了<script>标签的defer属性。
  • <script>标签的async属性也可以打开，这时只要加载完成，渲染引擎就会中断渲染立即执行。执行完成后，再恢复渲染。

# 5 ES6模块与CommonJS模块的差异

参考：阮一峰的ES6---Module的加载实现 (opens new window) 带有type="module"的<script>所引进来的是一个JS模块：

  1. CommonJS模块输出的是一个值的拷贝，ES6模块输出得失值的引用；
  2. CommonJS模块是运行时加载，ES6模块是编译时输出接口。
  3. CommonJS 模块的顶层this指向当前模块，ES6 模块的顶层的this指向undefined
  4. CommonJS模块输出的是值的拷贝，require命令第一次加载该脚本，就会执行这个脚本，然后在内存生成一个对象，也就是说一旦输出一个值，模块内部的变化就影响不到这个值。要更新需要再次require。
  5. js引擎对脚本静态分析的时候，遇到模块加载命令import，就会生成一个只读引用。等到脚本真正执行时，再根据这个只读引用，到被加载的那个模块里面去取值。原始值变了，import加载的值也会跟着变。因此，ES6 模块是动态引用，并且不会缓存值，模块里面的变量绑定其所在的模块。

# 6 移动端适配方案

  1. viewport 适配：计算scale = screen.width / WIDTH，再在meta标签中设置viewport的initial-scale属性为计算出来的scale;
  • 缺点：边线问题；等比缩放显示效果可能不好
  1. vm 适配（部分等比缩放）：计算出一个缩放的css变量值--width为 100 / WIDTH，单位为vm，在css中就可以直接使用
  • 对于需要等比缩放的元素，CSS使用转换后的单位
  • 对于不需要缩放的元素，比如边框阴影，使用固定单位px
  • 缺点：CSS中大量使用表达式，CSS表达式的执行需跳出CSS树的渲染，有性能问题
  1. rem 适配：设置html节点的fontSize为(100 * screen.width / WIDTH) + 'px'，假设设计稿为40px，则在使用时设置fontSize为0.4rem即可
  • 对于需要等比缩放的元素，CSS使用转换后的单位
  • 对于不需要缩放的元素，比如边框阴影，使用固定单位px
  • 为什么不设置 fontSize = 1px？这样就可以写 40 rem 了，更方便？ 主要是因为浏览器对字体大小有限制，设为 1px 的话，在浏览器中是失效的，会以 12px（或者其他值） 做一个计算 , 就会得到一个很夸张的结果，所以可以把 html 写的大一些
  1. 对于元素的布局，使用弹性布局。哪些页面需要做适配？
  • 页面中的布局是栅格化的

# 7 文档声明的作用

doctype 是 HTML5 标准的网页声明，<!DOCTYPE >必须声明在html文档的第一行，来告知浏览器的解析器用什么文档标准解析这个文档，不同的渲染模式会影响浏览器对于 CSS代码甚至javascript脚本的解析。 严格模式和混杂模式都是浏览器的呈现模式，浏览器究竟使用混杂模式还是严格模式呈现页面与网页中的DTD（文件类型定义）有关，DTD里面包含了文档的规则。

  • 严格模式：又称标准模式，是指浏览器按照W3C标准来解析代码，呈现页面
  • 混杂模式：又称为怪异模式或者兼容模式，是指浏览器按照自己的方式来解析代码，使用一种比较宽松的向后兼容的方式来显示页面。

常见声明：

  • HTML5
    <!DOCTYPE html>
    1
  • HTML 4.01 Strict 该 DTD 包含所有 HTML 元素和属性，但不包括展示性的和弃用的元素（比如 font）。不允许框架集（Framesets）。
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
    1
  • HTML 4.01 Transitional 该 DTD 包含所有 HTML 元素和属性，包括展示性的和弃用的元素（比如 font）。不允许框架集（Framesets）。
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
    1

# 8 html语义化标签（语义化的理解）

HTML语义化：就是页面去掉样式或者加载失败的时候能够让页面呈现出清晰的结构。HTML5新增了好多语义化的标签，例如：header、footer、nav、menu、section、article等等，单单从字面上理解，就知道标签的含义。在写页面的时候，我们可以直接引用这些标签，不需要再用没有任何含义的div标签了，对于机器可以识别，对于开发人员很容易明白。这就是HTML语义化。

语义化的好处：

  • 为了在没有CSS的情况下，页面也能呈现出很好地呈现内容结构、代码结构
  • 用户体验：例如title、alt用于解释名词
  • 有利于SEO：利于被搜索引擎收录，更便于搜索引擎的爬虫程序来识别
  • 方便其他设备解析（如屏幕阅读器、盲人阅读器、移动设备）以意义的方式来渲染网页
  • 便于项目的开发及维护，使HTML代码更具有可读性

# 9 src和href的区别？

  • href：（全称：Hypertext Reference，超文本引用），它指向一些网络资源，建立和当前元素或者说是当前文档的链接关系。
  • 在加载它的时候，不会停止对当前文档的处理，浏览器会继续往下执行。通常在a、link等标签中使用。
<a href="https://www.taobao.com"></a>
<link href="reset.css" type="text/css" rel="stylesheet">
1
2
  • src：（全称：source, 资源）表示的是对资源的引用，它执行的内容会嵌入到当前标签所在的位置（我们通常称它们为 可替换元素）。
  • 由于src的内容是页面必不可少的一部分，因此浏览器在解析src时，会停下来对后续文档的处理，直到src的内容加载完毕。常用在img、iframe、video、audio、script等标签中。
  • 这也就是为什么我们建议js文件放在HTML文档的最后面。如果js文件放在了head标签中，可以使用window.onload()方法，实现js的最后加载。
    • window.onload()方法的介绍：参考地址：https://www.runoob.com/w3cnote/javascript-window-onload.html
<img src="img/logo.jpg">
<iframe src="login.html">
<script src="index.js">
1
2
3

总结：

  1. href用于建立当前页面与引用资源之间的关系（链接）；而src则会替换当前标签。
  2. 遇到href，页面会并行加载后续内容；而src则不同，浏览器需要加载完毕src的内容才会继续往下执行。

# 10 常用的图片格式和应用场景

  • JPEG/JPG
    • JPEG图片支持的颜色比较多，图片可以压缩，但是不支持透明
    • 一般用来保存照片等颜色丰富的图片
  • GIF
    • GIF支持的颜色少，只支持简单的透明，支持动态图
    • 图片颜色单一或者是动态图时可以使用gif
  • PNG
    • PNG支持的颜色多，并且支持复杂的透明，不支持动图
    • 可以用来显示颜色复杂的透明的图片
  • webp
    • 谷歌新推出的专门用来表示网页的一种格式
    • 它具有其他图片格式的所有优点，而且文件格式还很小。缺点：兼容性不好
  • base64
    • 将图片使用base64编码，这样可以把图片转换为字符，通过字符形式来引入
    • 一般都是需要和网页一起加载的图片才会使用base64
