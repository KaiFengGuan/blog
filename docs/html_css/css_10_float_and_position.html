对浮动与定位的理解

GKFTech 2022/4/19 CSS

CSS的定位机制有3种：普通流（标准流）、浮动、定位，这几种都会影响元素在页面中的布局。 标准流就不多说了，这里主要整理一下对浮动和定位的理解。

# 1 浮动 float

# 1.1 什么是浮动？

浮动会使元素会脱离标准普通流的控制，移动到其父元素中指定位置。它可以让任何盒子可以在一行排列，用浮动来布局。

selector {
    float: value;
}
1
2
3
value 描述
none 元素不浮动（默认值）
left 元素向左浮动
right 元素向右浮动

# 1.2 浮动的特点

  • 脱离标准流，不占位置，只有左右浮动，在包含框的左边或者右边

  • 浮动元素依旧位于包含框之内，但是不会超过其内边距

  • 如果上一个元素浮动则会与其在顶部对齐，如果上一个元素是标准流则上下排列

  • 浮动元素不会互相重叠，不会上下浮动

  • 行内元素浮动后会变为块级元素

  • 浮动的框可以向左或者向右移动，直到他的外边缘碰到包含框或另一个浮动框的边框为止

    • 浮动元素的外边缘不会超过其父元素的内边缘
    • 行内元素浮动后会变为块级元素

# 1.3 清除浮动

# 为什么要清除浮动？

由于浮动元素不占用原文档流的位置，所以它会对后面的元素排版产生影响，为了解决这些问题，此时就要在该元素中清除浮动

准确来说，不是清除浮动而是清除浮动后造成的影响

# 其清除浮动的本质

清除浮动主要为了解决父级元素因为子级浮动引起内部高度为0 的问题

父盒子没有给高度就不会撑开盒子

# 如何清除浮动

（1）给受影响的元素添加clear属性

selector {
    clear: value;
}
1
2
3
value 描述
both 同时清除左右两侧浮动的影响
left 不允许左侧有浮动元素（清除左侧浮动的影响）
right 不允许右侧有浮动元素（清除右侧浮动的影响）
none 默认值。允许浮动元素出现在两侧。
inherit 规定应该从父元素继承 clear 属性的值。

（2）触发父级元素的BFC

将浮动元素的父元素触发BFC，如设置属性overflow: hidden;。

  • 优点：代码简洁

  • 缺点： 内容增多时候容易造成不会自动换行导致内容被隐藏掉，无法显示需要溢出的元素。

（3）浮动元素的父级div定义 伪元素::after

相当于在父级后面添加一个空的元素，然后给这个空的元素设置clear: both;。

.father::after {
    content: '';
    display: block;
    clear: both;
}

# 2 定位 position

元素的定位属性主要包括定位模式和边偏移两个部分：

  • 边偏移：元素相对于其父元素某条边的偏移量

    • 四个值：top、bottom、left、right

  • 定位模式：

    • static：自动定位（默认定位方式）
    • relative：相对定位，相对于其原文档流的位置进行定位
      • 不脱离文档流，不影响其他元素的布局
    • absolute：绝对定位，相对于其上一个已经定位的父元素进行定位
      • 脱离文档流，完全不占原位置，如设置偏移量会影响其他元素的定位
      • 绝对定位是将元素依据离自己最近的已经定位（非static）的父元素（祖先）进行定位；若父级元素一个都没有定位则以根元素（html）定位。
      • 应用：盒子水平垂直居中，子绝父相（首先left：50%;定位到父盒子的一半大小，然后自己的外边距负的一半值就可以margin-left）。
    • fixed：固定定位，相对于浏览器窗口进行定位
