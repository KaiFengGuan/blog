主要总结两栏自适应布局、三栏布局（圣杯布局和双飞翼布局）、粘连布局这几种。

# 1 两栏自适应布局

<div class="parent">
  <div class="left" style="background-color: lightblue;">
    <p>left</p>
  </div>
  <div class="right"  style="background-color: lightgreen;">
    <p>right</p>
    <p>right</p>
  </div>     
</div>

# 1.1 float布局 + BFC

如果是普通的两列布局，浮动+普通元素的margin便可以实现，但如果是自适应的两列布局，利用float+overflow:hidden便可以实现，这种办法主要通过overflow触发BFC，而BFC不会重叠浮动元素。具体代码如下：


 


 



 


.parent {
  overflow: hidden;
}
.left {
  float: left;
  margin-right: 20px;
}
.right {
  overflow: hidden;
}

# 2 三栏布局

三栏布局常用的情况是：中间列自适应宽度，旁边两侧固定宽度。

# 2.1 圣杯布局

圣杯布局是中间盒子优先渲染，两边的盒子框子宽度固定不变，中间部分自动填充整个区域，即使页面宽度变小，也不影响我们的浏览。

实现过程：

  • 在container中的三列设为浮动和相对定位(后面会用到)，center要放在最前面；
  • 三列的左右两列固定宽度100px，中间部分center设置100%撑满，这样因为浮动的关系，center会占据整个container，左右两块区域被挤下去了；
  • 设置left的 margin-left: -100%;，让left回到上一行最左侧；
  • 但这会把center给遮住，所以这时给外层的container设置 padding-left: 120px;和padding-right: 120px;，给left和right空出位置，多出的20作为空隙，可以不留；
  • 这时left并没有在最左侧，因为之前已经设置过相对定位，所以通过 left: -120px; 把left拉回最左侧；
  • 同样的，对于right区域，设置 margin-left: -120px; 把right拉回第一行，这时右侧空出了120px的空间；
  • 最后设置 right: -120px; 把right区域拉到最右侧。

<div class="container">
  <div class="center">
    <p>这里是中心内容</p>
  </div>
  <div class="left"></div>
  <div class="right"></div>
</div>

# 2.2 双飞翼布局

圣杯布局和双飞翼布局达到的效果基本相同，都是侧边两栏宽度固定，中间栏宽度自适应。 主要区别就是采用不同的解决方法来解决中间部分被遮挡的问题：

  • 圣杯布局是在父元素上设置了padding-left和padding-right，在给左右两边的内容设置position为relative，通过左移和右移来使得左右两边的内容得以很好的展现；
  • 双飞翼布局在圣杯布局基础上进一步优化，解决了圣杯布局错乱问题，实现了内容与布局的分离。而且任何一栏都可以是最高栏，不会出问题。具体是在主列center这个div中再加了一个div来放置内容，再给这个新的div设置margin-left和margin-right 。

双飞翼布局实现：

  • left、center、right三种都设置左浮动
  • 设置center宽度为100%
  • 设置负边距，left设置负边距为100%，right设置负边距为自身宽度
  • 设置content的margin值为左右两个侧栏留出空间，margin值大小为left和right宽度

<div class="container">
  <div class="center">
    <div class="content">
      <p>这里是中心内容</p>
    </div>
  </div>
  <div class="left"></div>
  <div class="right"></div>
</div>

# 3 粘连布局

# 3.1 粘连布局特点

  • 有一块内容<main>，当<main>的高度足够长的时候，紧跟在<main>后面的元素<footer>会跟在<main>元素的后面；
  • 当<main>元素比较短的时候(比如小于屏幕的高度),我们期望这个<footer>元素能够”粘连”在屏幕的底部。

# 3.2 实现步骤

  • footer必须是一个独立的结构，与wrap没有任何嵌套关系；
  • wrap区域的高度通过设置min-height，变为视口高度；
  • footer要使用margin为负来确定自己的位置；
  • 在main区域需要设置 padding-bottom。这也是为了防止负 margin 导致 footer 覆盖任何实际内容。

<div id="wrap">
  <div class="main">
    main <br />
    main <br />
    main <br />
  </div>
</div>
<div id="footer">footer</div>
</body>
