Vue3更新了什么？

GKFTech 2022/4/3 Vue3

# Vue3新特性

  1. Composition API
    • setup、setup语法糖
    • SFC CSS 变量
    • reactive、ref 与 toRefs
    • 生命周期钩子（向下兼容，更加语义化）
    • watch 与 watchEffect（指定侦听数据源 / 自动收集依赖）
    • teleport内置组件、suspense异步组件
  2. 自定义hooks，代码功能更集中和更明确
  3. 响应式（defineProperty => Proxy）
  4. Teleport：在A组件内部使用B组件, 又希望渲染的 DOM 结构不嵌套在A组件的 DOM 中
  5. Suspense：异步组件在默认情况下是可挂起的。使用 <Suspense>包裹异步组件，加载状态将由 <Suspense> 控制，组件自身的加载、错误、延迟和超时选项都将被忽略。
  6. Fragment：template下支持多个根节点
  7. 更好的Tree-Shaking：

# Vue3做的优化

  • 源码优化
    • 更好的代码管理方式：monorepo。根据功能将不同的模块拆分到 packages 目录下面不同的子目录中。模块拆分更细化，职责划分更明确，模块之间的依赖关系也更加明确，开发人员也更容易阅读、理解和更改所有模块源码，提高代码的可维护性。
    • TypeScript支持
  • 性能优化
    • 源码体积优化：首先，移除一些冷门的 feature（比如 filter、inline-template 等）；其次，引入 tree-shaking 的技术，减少打包体积。
    • 数据劫持优化：使用 Proxy API 做数据劫持
    • 编译优化：对静态模板的分析，编译生成 Block tree（基于动态节点指令切割的嵌套区块，区块内部节点结构是固定的）， 更新性能由与模版整体大小相关提升为与动态内容的数量相关
  • 语法 API 优化：Composition API
    • 优化逻辑组织：逻辑关注点集中
    • 优化逻辑复用：mixins => hook 函数，避免命名冲突和数据来源不清晰的问题

# setup语法糖

  • 子组件无需手动注册
  • 不需要返回方法和属性
  • 支持props、context API（useAttrs，useSlots，defineProps，defineEmits）
